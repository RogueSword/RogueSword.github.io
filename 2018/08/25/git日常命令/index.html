5. git pull详细

  • 定义： 从另一个存储库或本地分支获取并集成（整合）。取回远程某个分支的更新，再与本地的指定分支合并
  • 语法：git pull [options] [[…]]
  • 人话：git pull <远程主机名> <远程分支名>:<本地分支名>

6. git pull 和 git merge的区别

  • git fetch 是指从远程获取最新版本到本地，不会自动合并
  • 在实际使用中，git fetch更安全一些，因为在merge前，我们可以查看更新情况，然后再决定是否合并。
